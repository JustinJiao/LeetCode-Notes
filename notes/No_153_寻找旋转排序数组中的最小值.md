# LeetCode No.153 寻找旋转排序数组中的最小值

## 题目链接:https://leetcode.cn/problems/find-minimum-in-rotated-sorted-array/description/

## 题目描述

已知一个长度为 n 的数组，预先按照升序排列，经由 1 到 n 次 旋转 后，得到输入数组。例如，原数组 nums = [0,1,2,4,5,6,7] 在变化后可能得到：
若旋转 4 次，则可以得到 [4,5,6,7,0,1,2]
若旋转 7 次，则可以得到 [0,1,2,4,5,6,7]
注意，数组 [a[0], a[1], a[2], ..., a[n-1]] 旋转一次 的结果为数组 [a[n-1], a[0], a[1], a[2], ..., a[n-2]] 。

给你一个元素值 互不相同 的数组 nums ，它原来是一个升序排列的数组，并按上述情形进行了多次旋转。请你找出并返回数组中的 最小元素 。

你必须设计一个时间复杂度为 O(log n) 的算法解决此问题。
 

## 解题思路
- ** 方法 1 ** ：二分查找
由于数组是旋转排序数组，我们可以利用二分查找来找到最小元素。具体步骤如下：
1. 初始化两个指针left和right，分别指向数组的开头和结尾。
2. 当left < right时，计算中间位置mid = left + (right - left) // 2。
3. 比较nums[mid]和nums[right]的值：
   - 如果nums[mid] > nums[right]，说明最小元素在右半部分，更新left = mid + 1。
   - 否则，说明最小元素在左半部分，更新right = mid。
4. 最后，left指向的就是最小元素的位置，返回nums[left]。
时间复杂度`O(log n)`  
空间复杂度：`O(1)`

# 卡玛 109 多余的边二

## 题目链接: https://kamacoder.com/problempage.php?pid=1182

## 题目描述
题目描述
有一种有向树,该树只有一个根节点，所有其他节点都是该根节点的后继。该树除了根节点之外的每一个节点都有且只有一个父节点，而根节点没有父节点。有向树拥有 n 个节点和 n - 1 条边。如图： 





现在有一个有向图，有向图是在有向树中的两个没有直接链接的节点中间添加一条有向边。如图：




输入一个有向图，该图由一个有着 n 个节点(节点编号 从 1 到 n)，n 条边，请返回一条可以删除的边，使得删除该条边之后该有向图可以被当作一颗有向树。

输入描述
第一行输入一个整数 N，表示有向图中节点和边的个数。 

后续 N 行，每行输入两个整数 s 和 t，代表这是 s 节点连接并指向 t 节点的单向边

输出描述
输出一条可以删除的边，若有多条边可以删除，请输出标准输入中最后出现的一条边。
输入示例
3
1 2
1 3
2 3
输出示例
2 3
## 解题思路
- ** 方法 1 ** ：并查集
使用并查集。
主要是找入度为2的点，然后判断是否有环。
分为三种情况：
1. 有入度为2的点，并且有环，删除入度为2的点中形成环的边
2. 有入度为2的点，没有环，删除入度为2的点中最后出现的边
3. 没有入度为2的点，有环，删除环中的任意一条边（题目要求删除最后出现的边）
对于入度为2的点，我们可以先遍历一遍边，记录每个节点的入度，如果发现某个节点入度为2，则记录下这两条边。然后我们再使用并查集来检测环。也就是先假设删除入度为2的点中最后出现的边，然后进行并查集的合并操作。如果在合并过程中发现有环，则说明是第一种情况，返回入度为2的点中形成环的边。否则，说明是第二种情况，返回入度为2的点中最后出现的边。如果没有入度为2的点，则直接使用并查集检测环，返回环中的任意一条边（题目要求删除最后出现的边）。
对于没有入度为2的点的情况，我们可以直接使用并查集来检测环。在合并过程中，如果发现有环，则记录下形成环的边，最后返回最后出现的边。
时间复杂度：`O(N * α(N))`，其中 N 是边的数量，α 是反阿克曼函数。
大约等于线性时间复杂度。也就是 `O(N)`。
空间复杂度：`O(N)`，用于存储并查集的父节点和秩信息。
